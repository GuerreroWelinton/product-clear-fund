# Flujo de ramas y pull requests

## Ramas

```text
feature/FXX-descripcion
fix/FXX-descripcion
chore/descripcion
```

## Pull request

Debe incluir:

- Feature y requisitos implementados.
- Migraciones.
- Capturas de UI.
- Pruebas añadidas.
- Riesgos y rollback.
- Confirmación de autorización, auditoría y transacciones.
- Enlace a `acceptance.md`.

No mezclar varias features grandes en un mismo PR.
